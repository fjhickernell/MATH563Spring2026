Using Data Wisely

  • Collect the data

  • Explore the data

  • Choose a model / procedure

  • Draw conclusions

The data have several roles

  • Data collection → what data even exist

  • Exploratory Data Analysis (EDA) → learn from the data

  • Model selection → choose a procedure

  • Data snooping 🚫 → reuse data to chase results

👉 validity starts before analysis

Data collection

  • How are data obtained?

  • Design matters:

    • random sampling

    • random assignment

    • observational vs experimental

  • Issues:

    • bias

    • confounding

    • missingness

👉 No analysis fixes bad data

Exploratory data analysis (EDA)

  • Look first, decide later

  • Plots, summaries, transformations

  • Goals:

    • structure

    • anomalies

    • plausible directions

👉 hypothesis-generating, not testing

Model selection (broad view)

  • Choose among

    • probability models

    • regression models

    • test statistics

    • transformations

    • distributions / assumptions

  • Criteria:

    • fit

    • simplicity (parsimony)

    • interpretability

👉 choose a procedure for inference

Model selection: a subtle point

  • Selection uses the data

  • Then we perform inference as if the model were fixed

  • Consequence: uncertainty is underestimated

👉 p-values too small, confidence/prediction intervals too narrow

Data snooping 🚫

  • What it is

    • Trying many things on the same data

    • Reporting what “works”

  • Consequence: Type I error is inflated beyond the nominal level \(\alpha\)

👉 false discoveries become likely

Avoiding the data snooping trap

  • Separate roles:

    • exploration / training

    • validation

    • test

  • Or

    • pre-specify procedures

    • use cross-validation

👉 exploration ≠ confirmation

Takeaway

  • collection → foundation

  • EDA → understand

  • selection → decide

  • snooping → distort

👉 problems early propagate forward

Comments on Project Presentations

“Anything worth doing is worth doing well.” — source unknown

In general

  • You were excited about your topics
  • I learned a lot
  • Good stories were told
  • Motivation was clear
  • Slides were of high quality
  • The time limit was honored
  • There were clear explanations of the strengths and limitations of the article or simulation study

“Anything worth doing is worth doing badly.” — source unknown

May this experience make your next presentation even better. For some presentations

  • Mathematical notation was not typeset well, i.e., ugly
  • Mathematical notation was not clearly defined
  • The slides were too full
  • Presenters did not explain well what was on their slides
  • It was unclear what part of the presentation was a summary of the article, what was a critque, and what was the presenter’s own work
